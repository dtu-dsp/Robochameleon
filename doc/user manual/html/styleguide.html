Coding conventions

Overview

Nomenclature

Files

Unit names are written UnitName_vN, where UnitName is the name in CamelCase with capital first letter and N is the version number. Spell out all but the most commonly understood acronyms: e.g. EDFA is fine, NlCh is not.

Base unit names are all in lower case.

We're more flexible about utility functions, do whatever you feel like????

Properties

Properties should be in camelCase, with lower-case first letter. Again, spell out all but the most commonly used acronyms. Acronyms are in capital letters.

Modulation formats

In all cases, the order goes after the type (e.g. QAM16, NOT 16QAM). If the order is specified separately, it is called 'M'. This is to avoid issues that arise when file/function names begin with numbers. The order is the number of constellation points, not the number of bits per symbol.

We have agreed on the following official names for modulation formats - use whenever possible:

  1. ASK: amplitude shift keying. Also commonly called "OOK," "NRZ," or "PAM." Do not use either OOK or NRZ in your code. "PAM" is sometimes acceptable (cross-compatible with other parts of the code). Really, though, OOK should be ASK2, PAM4 should be ASK4, etc.
  2. Polybinary: n-binary, including duobinary. This can be Polybinary2, Polybinary3, etc.
  3. PSK: phase shift keying. The angle of the first constellation point is always zero, so QPSK should be treated as QAM4.
  4. QAM: square QAM. Other forms of QAM should have different names - e.g. constref.m uses 'star' for star QAM.

Use the utility function constref.m whenever possible to generate reference constellations.

Unit version numbering

Increment the version number when:

  1. Object properties have changed (been deleted, or meaning has ben altered. Adding a property doesn't require a new version).
  2. Main functionality has changed, such that the output signal is now meaningfully different from what it would have been absent your change. To safely (with respect to the git history) increment the version, use increaseClassVersion.m

Displaying information to the user

Plotting

Enable/disable plotting within a unit using unit::draw. This is a boolean variable, where 1/true means plot, 0/false means don't plot.

Don't set the absolute position of figures.

Keep in mind that signals vary greatly in length. Don't plot only the last 1000 samples of a signal, because it may not be 1000 samples long. Don't plot the whole signal in all cases either, because it may be 10^8 long.

Errors, warnings, and information

Use robolog.m to display errors, warnings and information.
